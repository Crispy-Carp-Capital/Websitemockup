## PART 6: Tech Stack Recommendation

### Frontend
- **Framework**: React 19 + TypeScript (modern, scalable)
- **State Management**: Zustand or Jotai (lightweight vs Redux overkill)
- **Charting**: Plotly.js or ECharts (professional, interactive)
- **UI Library**: MUI (Material-UI) or shadcn/ui + Tailwind CSS
- **Real-time Updates**: WebSocket + TanStack Query (React Query)
- **Tables/DataGrids**: TanStack Table (React Table) + custom styling

### Backend (Mock/Fake for now)
- **Framework**: FastAPI (Python) or Express.js (Node.js)
- **Real-time**: WebSocket with Socket.io or FastAPI WebSockets
- **Data**: In-memory cache (Redis) for quick prototyping
- **Mock Data**: Faker.js or Python Faker for synthetic training runs

### Deployment (Frontend only)
- Vercel (easiest), Netlify, or AWS S3 + CloudFront

## PART 8: Key Design Principles

1. **Trader-First**: Metrics and charts optimized for quick decision-making, not exhaustive reporting.
2. **Iterative Workflow**: 6-step wizard feels like a natural pipeline, not a form.
3. **Visual Feedback**: Real-time charts, progress bars, live alerts keep users engaged during training.
4. **Comparison Focus**: Always show model vs. benchmark; highlight wins.
5. **Transparency**: Show agent's "decisions" (action distribution, next predicted move) to build trust.
6. **Density**: Dashboard packs info without clutter; use tabs/collapsible sections if needed.
7. **Theming**: Offer dark mode by default for traders who monitor late into evening.
